# 대규모 시스템 설계 확장 계획

## 1. 캐시

현재 시스템은 **로컬 메모리 캐시**를 사용하고 있습니다.
하지만, **트래픽 급증**이나 **다중 서버 인스턴스 운영** 상황에서는 로컬 캐시만으로는 **일관성 문제**가 발생할 수 있습니다.

이를 해결하기 위해, **분산 캐시 솔루션(예: Redis, Memcached)** 을 도입하여 **모든 서버 인스턴스가 동일한 캐시 데이터를 참조**하도록 할 계획입니다.
이 방식은 **캐시 일관성 유지**, **캐시 적중률 향상**, 그리고 **시스템 성능 최적화**에 큰 도움이 됩니다.

---

## 2. 로드 밸런싱과 분산 서버

현재는 **단일 서버**에서 모든 요청을 처리하고 있으며, **WebSocket 연결**도 한 서버에 집중되어 있습니다.

앞으로는 **로드 밸런서를 도입**하여 트래픽을 여러 서버 인스턴스에 분산시킬 예정입니다.
이를 통해 **메인 서버와 웹소켓 서버의 부하 분산**, **시스템 확장성 향상**, **안정성 강화**를 동시에 달성할 수 있습니다.

---

## 3. LLM 기반 실시간 게임성 증가

향후 **LLM(대형 언어 모델)** 을 활용하여 **이벤트 기반 뉴스 및 트렌드 생성 기능**을 구현할 계획입니다.
이를 통해 사용자에게 **몰입감 있는 실시간 게임 경험**을 제공할 수 있습니다.

예를 들어, 특정 주식의 **급등락 시점에 자동으로 생성되는 뉴스**를 사용자에게 전달하여, **게임의 현실감**과 **참여도**를 동시에 높일 수 있습니다.

---

## 4. 회귀 모델 도입

향후 **주식 가격 예측**을 위해 **회귀 모델**을 도입할 예정입니다.
이를 통해 사용자가 **보다 전략적인 투자 결정**을 내릴 수 있도록 지원합니다.

구체적으로, **과거 주식 데이터를 기반으로 머신러닝 모델 학습 → 실시간 가격 예측 → 투자 인사이트 제공**의 흐름을 구현할 계획입니다.
이를 통해 **사용자에게 정교한 투자 전략**과 **게임의 깊이 있는 경험**을 제공할 수 있을 것입니다.
